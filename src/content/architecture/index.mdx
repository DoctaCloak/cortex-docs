# Architecture

Cortex is a three-tier application with a clear separation between frontend, backend, and storage.

## Key Design Decisions

1. **SQLite over Postgres** — Simpler deployment, zero infrastructure. Litestream provides durability via continuous replication to S3-compatible storage.

2. **ChromaDB for embeddings** — Embedded vector database. No separate service to manage.

3. **Haiku by default** — Cost efficiency. Most queries don't need Sonnet's reasoning power. Auto-escalation handles the exceptions.

4. **SSE over WebSockets** — Simpler, works through more proxies, sufficient for one-way event streaming.

5. **JWT over sessions** — Stateless authentication. No server-side session store needed.
